# Wiki Project Overview

A quiet, local‑first personal wiki: clean pages, fast search, zero noise.

## What It Is
This is a lightweight personal Wikipedia built on a Raspberry Pi.
Entries are static HTML pages, styled like classic wiki pages.
A tiny search index keeps it fast and simple.

## Why It Exists
I want one place to store what matters: projects, tools, and ideas—without clutter.
It’s designed to be readable now and useful years later.
No feeds, no noise, just a clean record.

## How It Works (3 Steps)
**Collect → Structure → Publish**
1) Collect notes locally (history logs first).
2) Structure them into entries with tags, categories, and links.
3) Publish as static pages with search.

## Privacy & Local‑First
Sensitive data never goes to GitHub.
The database stays local; only static pages are published.
Everything is designed to be quiet, safe, and controllable.

## Roadmap
- **MVP1/2:** layout, search, bilingual UI framework.
- **MVP3:** real personal data, local SQLite, privacy filter.
- **MVP4:** smarter extraction and automation—only after scope is clear.

# Wiki 项目简介

一个安静的本地个人百科：干净页面、快速搜索、没有噪音。

## 这是什么
这是一个运行在树莓派上的轻量个人百科。
条目是静态 HTML 页面，风格接近经典百科。
用一个小型搜索索引保持快速与简单。

## 为什么要做
我想把重要的项目、工具和想法放在一个地方，不被杂音打断。
它既能当下阅读，也能长期回溯。
没有信息流，只有清晰记录。

## 怎么做（3 步）
**收集 → 结构化 → 发布**
1) 本地收集（先从历史记录开始）。
2) 结构化为条目，带标签、分类、关联。
3) 生成静态页面并提供搜索。

## 隐私与本地优先
敏感数据不上传 GitHub。
数据库只保留在本地，仅发布静态页面。
系统保持安静、安全、可控。

## 路线图
- **MVP1/2：**布局、搜索、双语 UI 框架完成。
- **MVP3：**引入真实数据，本地 SQLite，严格隐私过滤。
- **MVP4：**在范围明确后再做更自动化的抽取。
